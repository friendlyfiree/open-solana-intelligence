# OSI V2 — Architecture Blueprint (index)

**Design-and-documentation only.** These documents define the constitutional and technical contract for the next version of Open Solana Intelligence. **No production code, `index.html`, CSS, Edge Function, SQL, or Supabase state is changed by this stage.** Nothing here is implemented; implementation begins only after the entity model, state machines, permissions, voting model, AI Pack trust model, memo spec, and migration plan agree with each other and the decision register is signed off.

**Revision note:** this set was revised after product-owner and security review. Key locked decisions now baked in across all documents: the **hybrid signature model** (individual analyst decisions are `signMessage` + server-verified receipts; public governance outcomes are Solana Memo anchored — D15), the **Stage-5 write gate** (`OSI_V2_WRITES_ENABLED` stays false until replay protection and receipt authenticity are verified — D14), **typed FK-backed review tables** (not one polymorphic `reviews` table), **Report/Wire versioning** (immutable versions, mutable headers), and **public analyst attribution** (D16). See `OSI_V2_OPEN_DECISIONS.md` for the full D1–D16 register.

**Second revision (remaining-inconsistency pass):** promoted three implied structures to first-class tables — `analyst_application_versions` (immutable, versioned applications; reviews target an exact version), `ai_pack_owner_feedback` (advisory, uncounted owner feedback, never in `ai_pack_reviews`), and `ai_pack_version_evidence` (per-layer evidence manifest with three scope hashes). Also: **Challenge targets are now real typed FKs** (exactly-one CHECK) with an `evidence_items` FK and **no stuck states** (every non-terminal state has a TTL/escalation); Case/Wire version submission is anchored as `CASE_REPORT_VERSION_SUBMITTED`/`WIRE_REPORT_VERSION_SUBMITTED` (class A); the report **published pointer** advances safely via `current_published_version_id` while resolutions stay bound to their exact selected version; and **every canonical event has exactly one transport class** (`AI_PACK_REJECTED`, `REPORT_REJECTED`, `CASE_SAFETY_BLOCKED` resolved to class A; `CHALLENGE_EXPIRED` to Sys — no `Sys/Memo` alternatives).

**Third revision (architecture-blocker pass):** the Constitution now itself states the Wire-Report standalone exception, Case-vs-Wire Report privacy, the **three** AI-Pack layers + per-layer manifest, that challenge submission alone does not pause sealing, and maintainer finalization aligned to the **locked D5 thresholds** (maintainer is an added gate only for resolution/winner, AI-Pack approval, and seal; Case initial open separately permits either its analyst threshold or a full double-gated maintainer, while initial rejection and all other listed outcomes remain analyst-quorum paths). The **resolution-selection chicken-and-egg is fixed** (the `case_resolutions` row is created in `selection_open` first, `winning_report_version_id` set once from the quorum tally, never invented by the maintainer). A mandatory **Stage-5 nonce/idempotency store `osi_nonces`** is defined (atomic single consumption; stateless checks forbidden). **Challenge** gains challenger exclusions, both-gate accept/reject, target-specific accepted consequences, and a **bad-faith review phase** (server-derived, quorum, immutable, with canonical events). Migration is made executable (`challenges_v2` name-collision fix, named `migration_crosswalk`/`migration_manual_queue`, a controlled backfill step, `security_invoker=true` views, corrected compat-view claim). Reversal transitions (withdraw/safety-lift/appeal/resume) are fully modeled with canonical events; the unsupported `unpublish` label is removed. Canonical event registry re-audited to **65 events, one class each**.

**Authoritative domain-table count: 32** — defined once in `OSI_V2_DOMAIN_MODEL.md §1`, referenced identically everywhere; no required domain entity is deferred "to implementation." **Mandatory infrastructure tables are named separately and are NOT inside that 32:** the Stage-5 security store `osi_nonces` and the migration-only `migration_crosswalk` + `migration_manual_queue`. The "32" is domain entities only; it never conceals security or migration infrastructure.

**Core shift:** from a mix of reports/bounties/votes/packs into a **Case-centered** protocol — Case is the primary entity, a bounty is an optional non-custodial reward pledge on a Case, Reports belong to exactly one Case, the Wire is a separate report-first lane, and every critical outcome needs ≥2 independent bounded-weight analysts plus (normal path) maintainer finalization.
