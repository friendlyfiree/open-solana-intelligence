# Code of Conduct

OSI is a public-good project about accountability. That starts with how we treat each other.

## Our standards

Everyone participating in this project's spaces (issues, pull requests, discussions, reviews) is expected to:

- Be respectful and assume good faith. Disagree with ideas, not with people.
- Keep criticism specific, technical, and actionable.
- Accept review and reversal gracefully; the product itself is built on the idea that any decision can be challenged with evidence.
- Respect privacy. Never post anyone's personal information, and never use project spaces to dox, harass, or target any individual, including subjects of investigations.
- Follow the product's own ethics in discussion: no unsupported accusations of guilt or crime against any person or entity. Cases speak through evidence and process, not through comment threads.

## Unacceptable behavior

- Harassment, discrimination, or personal attacks in any form.
- Publishing private information of any person without explicit permission.
- Using project spaces to advance scams, impersonation, or market manipulation.
- Deliberately submitting fabricated evidence, fake reviews, or misleading claims.
- Sustained disruption of reviews, discussions, or governance processes.

## Scope

This code applies in all project spaces and when representing the project publicly. It complements, and never overrides, the platform rules enforced in the product itself.

## Enforcement

Report conduct issues to the maintainer through a GitHub issue, or privately through a security advisory if the report itself is sensitive. Reports are reviewed promptly and confidentially. Consequences scale with severity, from a warning to removal from project spaces. Enforcement decisions concerning project spaces follow the same value the product does: proportionate, recorded, and open to appeal.
