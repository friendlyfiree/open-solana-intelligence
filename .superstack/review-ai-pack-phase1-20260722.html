Findings and applied fixes

criticalresolvedGeneration authorization could become stale between preparation, provider work, and commit.

Fix: repeat the public Case, exact stage, non-owner, analyst eligibility, and double-maintainer checks before provider use and again before storage.

case_row.visibility = 'public'
case_row.stage IN (accepted eligible stages)
case_row.submitted_by_wallet <> actor_wallet

highresolvedAn unstored evidence drift could allow review or approval of an obsolete artifact.

Fix: recompute the three exact manifest hashes in prepare and commit RPCs; return SQLSTATE 40001 and require regeneration if any layer differs.

highresolvedA promoted Wire header could make evidence from later Wire versions appear in the existing Case manifest.

Fix: accept only evidence explicitly bound to the Case. Wire promotion already copies the exact promoted version evidence to case_evidence_links.

highresolvedOlder review states could remain nonterminal after replacement, and the constraint incorrectly required approval metadata on every superseded version.

Fix: implement accepted supersession transitions, link predecessors to the exact replacement, and allow approval metadata only as an all-null or all-present immutable group.

mediumresolvedAn expired provider reservation could block future generation indefinitely.

Fix: close expired reservations atomically during the next signed reservation attempt and leave provider spend fields null when the old isolate cannot reconcile them.

mediumresolvedThe production smoke used an invalid bare capabilities request and an undefined rollout-mode variable.

Fix: remove the malformed request and record the exact fixed mode deploy-disabled.

mediumopen validationThis workstation lacks Supabase CLI, Docker, Bash, and Deno.

Required before merge or rollout: the existing CI job must pass the following checks.

supabase db reset --local --no-seed
supabase db lint --local --level error --fail-on error
